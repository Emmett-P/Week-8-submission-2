1) The Gaming Room wanted to expand their Android drawing game, Draw It or Lose It, to work across multiple platforms like iOS, Windows, and web browsers while supporting multiple simultaneous players securely.
2) I did a good job clearly explaining each recommendation in the design document and tying them back to the client's actual requirements.
3) Working through the design document first made it easier to spot potential problems with things like memory and multi-user support before they became issues in the code.
4) I would go back and add more detail to the Operating Systems Architectures section
5) I focused on making the game accessible from any device without requiring a separate download, because designing without the user in mind usually means building something that technically works but does not actually solve the problem.
6) I started with the client's requirements and worked outward from there to pick the architecture, storage, and security choices that best supported them, which is an approach I would use again for any similar project.
